# On-device NN scores not written to EXIF — `USE_PERCENTAGE` build-flag mismatch

**Found:** 2026-06-19, while analysing why model #2's predictions weren't reaching
the Wildlife Watcher website. **Impact:** the device writes auto-exposure telemetry
(MakerNote) and the deployment ID, but **no NN class scores** (UserComment), so the
website has nothing to ingest.

## Symptom

Images captured with a model loaded contain:
- `MakerNote (0x927C)` = AE registers, e.g. `"76, 0, 64, 68, Y"` ✅
- Deployment ID (`0xF200`) ✅
- **UserComment (0x9286) — absent** ❌ (should be e.g. `"not rat: 88%; rat: 12%; "`)

## Root cause — two coupled flags set inconsistently

`image_task.c` builds the UserComment under **`ENABLE_EXIF_CONFIDENCE`** (defined,
line 87) by calling `cv_get_confidence_data()` and reading `class_count` / `labels`
/ `confidence_percent`:

```c
// image_task.c (Path A)
cv_get_confidence_data(&confidence_data);
has_confidence_data = (confidence_data.class_count > 0 && confidence_data.class_count <= MAX_CLASSES);
...
exif_input.user_comment = (has_confidence_data && user_comment[0] != '\0') ? user_comment : NULL;
```

But everything that **produces** that data in `cvapp.cpp` — the softmax→percentage
computation that fills `g_last_confidence_data` (lines ~810–903) **and** the getter
`cv_get_confidence_data()` itself (lines ~931–939) — is gated behind a *different*
flag, **`USE_PERCENTAGE`**, which is **commented out**:

```c
// cvapp.h:33
//#define USE_PERCENTAGE     ← disabled
```

With `USE_PERCENTAGE` off, the confidence data is never computed, so `class_count`
stays 0 → `has_confidence_data` is false → `user_comment` is `NULL` → no UserComment
tag is emitted. (AE telemetry is unaffected; it's gated by `EXIF_MAKER_NOTES`, which
is defined — hence MakerNote is present.)

## Fix

Enable `USE_PERCENTAGE` so it is defined alongside `ENABLE_EXIF_CONFIDENCE`,
rebuild, and reflash:

```c
// cvapp.h:33
#define USE_PERCENTAGE
```

Then the device runs softmax on the model output and writes
`UserComment = "<label>: <pct>%; …"` using the labels loaded from the SD-card
manifest (`manifest/labels.txt` + `MOD0000X.tfl`, generated by the website's
"Prepare SD Card").

**Recommended:** couple the two flags so they can't drift again — e.g. have
`ENABLE_EXIF_CONFIDENCE` imply `USE_PERCENTAGE` (or `#error` if one is set without
the other), since `image_task.c`'s Path A cannot work without the `cvapp.cpp`
producer.

## Website side (for awareness)

The website already parses `UserComment` as `"label: value%;"` and is being extended
to (a) capture each model's label→species mapping at upload (`ai_models.label_map`),
and (b) reflect on-device predictions as taxa observations alongside the cloud
SpeciesNet run. Both depend on the device actually emitting UserComment — i.e. on
this flag fix.
