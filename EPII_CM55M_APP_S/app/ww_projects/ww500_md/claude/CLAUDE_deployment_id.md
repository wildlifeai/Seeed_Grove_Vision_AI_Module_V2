## The CONFIG.TXT file

The HX6538 processor needs to be placed in deep-power-down (DPD) when idle to save power. It reboots when necessary (e.g. when the image sensor detects motion). Certain state is saved in a file `CONFIG.TXT` before entering DPD and is restored shortly after booting.

Until recently, data stored in `CONFIG.TXT` was exclusively in the form of index, value pairs - one per line. The index values are defined in `OP_PARAMETERS_E` in `fatfs_task.h`

Recently I extended this to allow for a GPS location string to be saved and restored - the example is:

```
G 27°59'17.28" N 86°55'30.12" E 8848.86 Above
```
This is generated by code that includes `exif_gps_create_full_string()` and parsed with `exif_gps_parse_full_string()`

The `CONFIG.TXT` file is read by `load_configuration()` and written by `save_configuration()`

## Deployment ID

A 128-bit UUID is used with each 'deployment' of the camera device. The UUID is saved as a EXIF field in the JPEG images. Because `CONFIG.TXT` has only been able to manage index, value pairs the UUID has been saved as 8 16-bit values with indexes OP_PARAMETER_DEPLOYMENT_ID_CHUNK_1 - OP_PARAMETER_DEPLOYMENT_ID_CHUNK_8.

A string is generated by `fatfs_getDeploymentId()`. There is no "setter" - UUID values are written by setting `op_parameter[OP_PARAMETER_DEPLOYMENT_ID_CHUNK_1]` etc.

I would like to work towards saving the UUID as a single line in `CONFIG.TXT` in the same way as the GPS string. Perhaps it can be identified as `I ` as the first two characters.

To allow progressive testing of both the embedded application and the smart phone app (that generates the UUID) the OP_PARAMETER_DEPLOYMENT_ID_CHUNK_1 - OP_PARAMETER_DEPLOYMENT_ID_CHUNK_8 should be retained for the moment. When testing of the new code is finished we will remove these.

The app communicates with the HX6538 by sending lines through an I2C interface and the messages end up being processed as strings in `CLI-commands.c` We should add two commands to allow the app send and retrive UUID deployment ID string:

* setdid <string>
* getdid

You can follow the pattern in `CLI-commands.c` to add these commands. There will be a need for functions to parse `CONFIG.TXT` when it is loaded to extract the UUID, and to write it into the EXIF data.
